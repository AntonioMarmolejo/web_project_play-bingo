PROYECTO · 2025

BingoScan

Plataforma web y móvil para gestión de cartones de bingo. Carga por foto, PDF o digitación manual. Juego en tiempo real para múltiples jugadores.

React React Native Node.js Express.js MongoDB Socket.io Tesseract OCR Google Vision API JWT Auth PDF.js Cloudinary
01
DOCUMENTO PRINCIPAL Requerimientos
📸
Carga de Cartones
  • Fotografiar hoja física con cámara del dispositivo
  • Cargar PDF con múltiples tablas de bingo
  • Digitación manual de números por tabla
  • OCR automático para extraer números de imagen
  • Confirmación/corrección de datos extraídos
  • Soporte para múltiples cartones por partida
🎮
Juego en Tiempo Real
  • Sala de juego con código único de acceso
  • Marcado automático al cantar número
  • Detección automática de bingo ganador
  • Historial de números cantados
  • Múltiples jugadores simultáneos
  • Roles: Administrador / Jugador
👤
Gestión de Usuarios
  • Registro y autenticación con JWT
  • Perfil de usuario con historial de partidas
  • Creación y administración de salas
  • Invitación de jugadores por enlace
⚙️
No Funcionales
  • App Web + App Móvil (iOS / Android)
  • Latencia < 200ms en eventos de juego
  • Disponibilidad 99.9% uptime
  • Soporte offline parcial (ver cartones)
  • OCR con precisión ≥ 92%
  • Escalable a 500 jugadores por sala

REQUERIMIENTOS → HISTORIAS Historias de Usuario
US-01
Como jugador, quiero fotografiar mi hoja de bingo física para que el sistema extraiga automáticamente los números de mis cartones sin tener que digitarlos.
ALTA
US-02
Como jugador, quiero cargar un PDF con varias tablas de bingo para que el sistema identifique y registre cada cartón individualmente.
ALTA
US-03
Como jugador, quiero digitar manualmente los números de un cartón específico cuando la foto no tiene buena calidad.
ALTA
US-04
Como jugador, quiero corregir los números que el OCR extrajo incorrectamente antes de confirmar mi cartón.
ALTA
US-05
Como administrador, quiero crear una sala de juego y obtener un código para compartir con los participantes.
ALTA
US-06
Como administrador, quiero cantar números aleatoriamente y que todos los cartones de los jugadores se marquen en tiempo real automáticamente.
ALTA
US-07
Como sistema, quiero detectar automáticamente cuando un cartón completa bingo y notificar al administrador y jugadores.
MEDIA
US-08
Como jugador, quiero ver el historial de números cantados durante la partida para revisar si me perdí alguno.
MEDIA
US-09
Como usuario, quiero ver el historial de mis partidas anteriores y los cartones que usé.

04
ARQUITECTURA Diseño Técnico
CAPA CLIENTE · Frontend
React Web App React + Vite + TailwindCSS Interfaz web responsiva. Acceso desde cualquier browser.
React Native App Expo · iOS + Android App móvil. Acceso a cámara para escaneo de cartones.
Estado Global Zustand / Redux Toolkit Manejo de sesión, sala activa y cartones del jugador.
Socket Client socket.io-client Conexión en tiempo real con el servidor de juego.
⬇   HTTP REST + WebSocket WSS   ⬇
CAPA API · Backend
API REST Node.js + Express.js Autenticación, usuarios, cartones, salas y partidas.
Servidor WebSocket Socket.io Eventos de juego en tiempo real: números, marcados, bingo.
Auth Middleware JWT + bcrypt Validación de tokens en cada petición protegida.
OCR Service Google Vision API / Tesseract.js Extracción de números desde imagen o PDF de cartones.
⬇   MongoDB Driver + Mongoose   ⬇
CAPA DATOS · Persistencia
MongoDB Atlas Base de datos principal Colecciones: users, rooms, cartones, games, numberHistory
Cloudinary Almacenamiento de imágenes Fotos y PDFs subidos por usuarios. CDN global.
Redis (opcional) Cache + Pub/Sub Estado de sala en memoria para respuesta ultra-rápida.
⬇   Deploy   ⬇
INFRAESTRUCTURA · Deploy
Vercel / Netlify Frontend Web Deploy automático desde GitHub. CDN global.
Railway / Render Backend Node.js Servidor con soporte para WebSockets persistentes.
Expo EAS App Móvil Build y distribución para iOS App Store y Google Play.

ESQUEMA · Colecciones MongoDB

👤
users
  • _id, name, email, password
  • role: [admin | player]
  • gamesHistory: [gameId]
  • createdAt, updatedAt
🏠
rooms
  • _id, code (único), adminId
  • players: [userId]
  • status: [waiting|active|ended]
  • currentGame: gameId
🎴
cartones
  • _id, userId, roomId
  • grid: [[5x5 números]]
  • markedNumbers: [nums]
  • sourceType: photo|pdf|manual
  • imageUrl (Cloudinary)
